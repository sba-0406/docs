# HR Question Moderation Flow

This flow covers how HR users review and manage AI-generated questions to ensure quality and relevance before candidates take an assessment.

## 1. Identifying Moderation Needs
- **Component**: `job-detail.ejs` (HR Dashboard)
- **Logic**: 
    - When a job is in `draft` or `active` status, the system checks its associated `Assessment`.
    - If any technical questions have `status: 'pending_review'`, a "Moderation Required" warning is displayed.
    - If there are `missingSkills` (skills detected in JD but not found in the bank), a specialized warning explains how to fill the gap using AI generation.

## 2. Reviewing AI-Generated Questions
- **Component**: `job-detail.ejs` -> Question Pool Section
- **UI**: Questions marked with an `AI GEN` tag and `Approve/Reject` buttons.
- **Action**: HR reads the question text and options generated by the AI.

## 3. Moderation Actions
- **Component**: `jobController.moderateQuestion`
- **Approve**: 
    - Sets the question status to `active` in the global `Question` bank.
    - This allows the question to be reused in other assessments.
- **Reject**:
    - Removes the question from the current assessment.
    - Sets the question status to `retired` in the global bank.

## 4. Assessment Status Update
- **Logic**:
    - After every moderation action, the `Assessment` status is re-evaluated.
    - If all questions are now `active` and no `missingSkills` remain, the assessment status changes from `pending_review` to `active`.
    - Only `active` assessments allow candidates to proceed past the application phase.

## 5. Regeneration & Fallback
- **Component**: `jobController.regenerateTechnicalAssessment`
- **Action**: HR can click "Regenerate All" to wipe the current pool and try again with new AI generation or updated skill configurations.
- **Reasoning**: Useful if the initial AI generation produced low-quality questions or if the Job Description was updated.
